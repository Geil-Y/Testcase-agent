# Review Pipeline

Clarification-first, confidence-routed, human-reviewable test case generation pipeline.

Replaces the old `analyze_and_plan → generate_case → self_check` flow.

## Architecture

```
Code = plumbing only.
Prompt = generation philosophy and semantic judgment.
JSON = source of truth.
HTML = human review view only.
```

## Flow

```
Input requirements
  → LLM-A Requirement Decomposer (prepare-clarification-review)
  → clarification_review.json + clarification_review.html
  → human edits clarification_review.json
  → validate-review
  → clarified_test_basis.json
  → LLM-B Case Intent Planner (prepare-intent-review)
  → case_intent_review.json + case_intent_review.html
  → human edits case_intent_review.json
  → validate-review
  → approved_case_plan.json
  → LLM-C Case Writer (generate-cases)
  → generated_cases.json
  → evaluate (hard-rule evaluation)
  → optional import-memory
```

## Commands

| Command | Input | Output |
|---|---|---|
| `prepare-clarification-review --input <req.json> --out <dir>` | Requirements JSON | `clarification_review.json` + `.html` |
| `validate-review --file <dir>/clarification_review.json` | Edited review | `clarified_test_basis.json` |
| `prepare-intent-review --run-dir <dir>` | Clarified test basis | `case_intent_review.json` + `.html` |
| `validate-review --file <dir>/case_intent_review.json` | Edited review | `approved_case_plan.json` |
| `generate-cases --run-dir <dir>` | Approved plan | `generated_cases.json` |
| `evaluate --run-dir <dir>` | Generated cases | Evaluation reports |
| `import-memory --run-dir <dir>` | Validated artifacts | SQLite Review Memory |

## Phase 1 Scope

In scope:
- Full clarification-first pipeline with placeholder LLM calls
- Confidence aggregation and color routing
- Deterministic pattern tag derivation
- Human review validation with reason codes
- Review Memory SQLite storage and retrieval
- Hard-rule evaluation integration
- End-to-end offline fixture testing

Out of scope for phase 1:
- Main UI integration
- Excel import
- Automatic human skipping
- Automatic prompt optimization
- Embedding search
- Code-driven automatic approve/reject
- Full replacement of existing evaluator logic
- `self_check` / repair stage

## Design Rules

- Review Memory is advisory, not authoritative. It may affect confidence,
  routing, and reviewer hints, but must not introduce expected behavior,
  thresholds, or case intents not supported by the current requirement.
- `pattern_tags` are deterministic memory indexes derived by code, never by
  LLM or direct human editing.
- HTML is generated by code from JSON. LLM never authors HTML.
- Human edits JSON directly in phase 1.
